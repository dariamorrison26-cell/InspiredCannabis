# Directives

SOP-style instructions written in Markdown.  
Each directive defines: **goals, inputs, tools/scripts, outputs, and edge cases.**

Written in plain language, as if briefing a competent teammate.

## Naming Convention
- `directive_name.md` — lowercase, underscores
- Each file is a self-contained SOP for a single workflow

## Structure Template
```markdown
# Directive: [Name]

## Goal
What needs to be accomplished.

## Inputs
What data/files/context is needed.

## Tools / Scripts
Which execution scripts to use.

## Outputs
What the deliverable looks like.

## Edge Cases & Learnings
Known constraints, API limits, gotchas discovered through self-annealing.
```

How It Works

Data Collection & Storage

The Outscraper API scrapes Google Business Profile pages for all 44 stores. The raw scraped data — including review date, star rating, reviewer name, review text, and owner response — is first written to a SQLite database. This database serves as the central data store and the single source of truth for all downstream outputs.

Each sync is incremental: only reviews newer than the last known review are fetched, avoiding duplicate data and minimizing API costs.

How Weekly Updates Are Triggered

Weekly updates are triggered automatically by GitHub Actions — a cloud-based automation service built into the code repository. Every Monday at 8:00 AM PT, GitHub spins up a virtual server, runs the sync pipeline, and shuts down. No human intervention is needed.

The weekly workflow performs these steps:

  1. Restores the database from the previous run
  2. Fetches new reviews from Google (via Outscraper) for all 44 stores
  3. Adds new reviews to the database
  4. Pushes updated data to the "All Reviews" and "Needs Attention" tabs in Google Sheets
  5. The "Online Reviews 2026" tab auto-recalculates — no code touches it

The pipeline can also be triggered manually at any time for on-demand refreshes.

Important: The pipeline only writes to the "All Reviews" tab (new rows). The "Online Reviews 2026" tab is never overwritten — it contains live formulas that auto-recalculate when new data is added.
